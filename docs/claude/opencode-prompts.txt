PROMPT #2: IMPLEMENT AI ORCHESTRATOR SERVICE
────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

🎯 GOAL: Build core service that manages parallel API calls to three AIs

Create the AIOrchestrator NestJS service that:

1. Takes a user prompt and forks it to Claude, ChatGPT, and Gemini in PARALLEL (not sequential)
2. Each AI gets its own system prompt so they behave authentically:
   - Claude: "thoughtful, nuanced, harmless, honest"
   - ChatGPT: "comprehensive, well-organized, practical examples"
   - Gemini: "balanced, multimodal-aware, solution-focused"

3. Uses max_tokens: 2048 for all three (prevents incomplete responses)
4. Returns all three responses even if one fails (Promise.allSettled)
5. Supports chat history (prior messages in conversation)

CRITICAL: The service must have:
- async generateChatResponse(context) → returns array of responses
- async analyzeDocument(docContent, question) → for commissioning mode
- System prompts defined as class property
- Each AI call wrapped in error handling
- Response format: { source, content, model, tokens }

Use @anthropic-ai/sdk for Claude, openai for ChatGPT, @google/generative-ai for Gemini.

PROMPT #3: IMPLEMENT AI ORCHESTRATOR CONTROLLER
────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

🎯 GOAL: Build API endpoints that the frontend calls

Create AIOrchestrationController with two endpoints:

1. POST /api/ai/chat
   - Input: { prompt, history?, sessionId? }
   - Calls AIOrchestrator.generateChatResponse()
   - Returns: { success, data: { responses: [...] } }

2. POST /api/ai/analyze-document
   - Input: { documentContent, userQuestion, fileName?, sessionId? }
   - Calls AIOrchestrator.analyzeDocument()
   - Returns: { success, data: { responses: [...] } }

Both endpoints should:
- Use @UseGuards(JwtAuthGuard)
- Validate inputs (throw BadRequestException if empty)
- Catch errors and throw InternalServerErrorException
- Return consistent response format

PROMPT #5: IMPLEMENT PRISMA SCHEMA
────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

🎯 GOAL: Define database structure for storing sessions, messages, and documents

Create Prisma schema with three models:

1. ChatSession
   - id: String @id @default(cuid())
   - userId: String
   - mode: Enum ('chat' | 'commissioning')
   - createdAt: DateTime @default(now())
   - updatedAt: DateTime @updatedAt
   - messages: ChatMessage[] (relation)
   - documents: Document[] (relation)

2. ChatMessage
   - id: String @id @default(cuid())
   - sessionId: String (FK)
   - userPrompt: String
   - claudeResponse: String?
   - chatgptResponse: String?
   - geminiResponse: String?
   - timestamp: DateTime @default(now())

3. Document
   - id: String @id @default(cuid())
   - sessionId: String (FK)
   - fileName: String
   - content: String (raw text from upload)
   - claudeAnalysis: String?
   - chatgptAnalysis: String?
   - geminiAnalysis: String?
   - uploadedAt: DateTime @default(now())

Add proper indexes, relations, and cascade delete where appropriate.

HOW TO USE THESE IN OPENCODE:

Option A: Sequential Prompts
1. Copy PROMPT #1, paste into OpenCode, execute
2. Copy PROMPT #2, paste into OpenCode, execute
3. Continue through PROMPT #10

Option B: Pipeline Agents (Advanced)
Create .opencode/agents/AGENTS.md:

```
agents:
  - name: scaffold
    prompt: "PROMPT #1"
    next: backend-service
  - name: backend-service
    prompt: "PROMPT #2"
    next: backend-controller
  - name: backend-controller
    prompt: "PROMPT #3"
    next: frontend-component
  - name: frontend-component
    prompt: "PROMPT #4"
    next: database
  - name: database
    prompt: "PROMPT #5"
    next: error-handling
```

Then execute: opencode agents run

Option C: Single Mega-Prompt (Fastest)
Copy the "COMPREHENSIVE MULTI-AI ORCHESTRATOR BUILD" prompt and paste it all at once.
